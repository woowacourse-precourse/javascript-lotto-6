# 허용되는 유형:
# feat    : 새로운 기능 추가
# fix     : 버그 수정
# docs    : 문서 변경
# style   : 코드 의미에 영향을 주지 않는 변경 (공백, 포맷팅, 세미콜론 누락 등)
# refactor: 버그를 수정하거나 기능을 추가하지 않는 코드 변경
# perf    : 성능을 향상시키는 코드 변경
# test    : 누락된 테스트 추가 또는 기존 테스트 수정
# build   : 빌드 시스템 또는 외부 종속성에 영향을 주는 변경 (예: gulp, broccoli, npm)
# ci      : CI 구성 파일과 스크립트 변경 (예: Travis, Circle, BrowserStack, SauceLabs)
# chore   : src나 test 파일을 수정하지 않는 기타 변경
# revert  : 이전 커밋을 되돌림
# ---
# 
# 기억할 점:
# - 현재 시제를 사용하세요 ("Add feature"가 아닌 "Added feature")
# - 명령어 형태를 사용하세요 ("Move cursor to..."가 아닌 "Moves cursor to...")
# - 첫 번째 줄을 72자 이내로 제한하세요
# - 첫 번째 줄 뒤에 이슈 및 풀 리퀘스트를 자유롭게 참조하세요
# - 문서만 변경하는 경우 커밋 제목에 [ci skip]을 포함하세요
# - 커밋 범위를 존중하세요: 커밋이 코드베이스의 특정 부분에 적용된다면 범위에서 이를 나타내세요
# 
# 예시:
# feat(파서): 배열 파싱 기능 추가
# fix(버튼): 버튼 색상 수정
# docs(리드미): 설치 지침 업데이트
